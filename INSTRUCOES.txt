# LançaEnsaio - Versão Unificada

Instruções para deploy e manutenção do sistema unificado (Irmãos e Irmãs).

## 1. Backend (Supabase Edge Function)

1. Arquivo principal: `supabase/functions/api/index.ts`
2. Configure as Secrets no Supabase:
   - `ORQUESTRA_SHEET_ID`: ID da sua planilha Google.
   - `GOOGLE_SERVICE_ACCOUNT_B64`: JSON da Service Account em Base64.
3. Deploy da Function:
   ```bash
   supabase functions deploy api --no-verify-jwt
   ```

## 2. Mobile (Expo)

1. Arquivos principais:
   - `mobile/app/(tabs)/index.tsx` (Lançar)
   - `mobile/app/(tabs)/alert.tsx` (Alerta)
   - `mobile/app/(tabs)/settings.tsx` (Configurações)
   - `mobile/app/setup.tsx` (Fluxo inicial)
2. Dependências:
   ```bash
   cd mobile
   npm install
   ```
3. Rodar localmente:
   ```bash
   npx expo start
   ```
4. Build APK (EAS):
   - Certifique-se de configurar `EXPO_PUBLIC_API_URL` no EAS.
   - `eas build -p android --profile production`

## 3. Funcionamento Unificado

- O app solicita Nome e Modo (Irmãos/Irmãs) na primeira vez.
- Não há login tradicional.
- A auditoria grava: `META APP=UNIFICADO TIPO={tipo} USER={nome}`.
- O sistema de "locks" foi removido para maior simplicidade (Append simples).
